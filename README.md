# Legal Document Classifier (Mini Project)

An end-to-end mini project for legal document classification with:

- Baseline model: BiLSTM (PyTorch)
- Strong model: Legal-BERT fine-tuning (Transformers)
- Dataset helper: download legal benchmark data and export to CSV
- Flask + HTML UI for inference

## Project structure

```text
legal_classifier/
  app.py
  requirements.txt
  README.md
  data/
    raw/
      sample_legal_docs.csv
      legal_scotus.csv           # generated by dataset downloader
  artifacts/                  # generated after training
  src/
    legal_classifier/
      __init__.py
      config.py
      data.py
      model.py
      train.py
      train_transformer.py
      predict.py
      download_dataset.py
  templates/
    index.html
  static/
    styles.css
```

## 1) Setup

```bash
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

pip install -r requirements.txt
```

## 2) Better dataset (recommended)

Download a legal benchmark dataset (LexGLUE -> SCOTUS):

```bash
python -m src.legal_classifier.download_dataset --dataset lex_glue --subset scotus --output data/raw/legal_scotus.csv
```

This creates `data/raw/legal_scotus.csv` with `text,label,split`.

## 3) Train model

### Option A: Strong model (recommended)

```bash
python -m src.legal_classifier.train_transformer --data_path data/raw/legal_scotus.csv --model_name nlpaueb/legal-bert-base-uncased --epochs 4 --batch_size 8
```

### Option B: Baseline model

```bash
python -m src.legal_classifier.train
```

### Artifacts created

For transformer:

- `artifacts/transformer_model/`
- `artifacts/model_type.json`
- `artifacts/label_to_id.json`
- `artifacts/id_to_label.json`
- `artifacts/classification_report.json`
- `artifacts/training_history.json`

For BiLSTM:

- `artifacts/model.pt`
- `artifacts/vocab.json`
- `artifacts/label_to_id.json`
- `artifacts/id_to_label.json`

## 4) Run the web app

```bash
python app.py
```

Then open: `http://127.0.0.1:5000`

The app auto-detects the model type from `artifacts/model_type.json`.

## 5) Use your own dataset

Use a CSV with columns:

- `text` (string)
- `label` (class name)

Example:

```csv
text,label
"This agreement is made between the parties...",contract
"The appellant challenges the lower court order...",court_order
```

Then train with:

```bash
python -m src.legal_classifier.train_transformer --data_path path/to/your.csv
```
